Macros calculados por comida y totales diarios
Base de datos nutricional usada: valores de composición por 100g en crudo (BEDCA/USDA). Se multiplican por la cantidad de cada alimento en cada mes/fase y se suman por toma. Los valores cocinados difieren pero la referencia siempre es en crudo como indica tu plan.
¿Cómo se calculan las cantidades?
⚠️ Versión anterior (lookup table): La v1 de la herramienta no calculaba — simplemente asignaba valores fijos por categoría de fase basándose en el historial. Útil como referencia pero no científico. Esta v2 añade cálculo real desde primeros principios.
✅ Esta versión (fórmulas reales): Calcula TDEE → objetivo calórico por fase → reparto de macros → cantidad de alimentos. Todo explicado y auditable aquí.
PASO 1 — Tasa metabólica basal (BMR)
Fórmula: Mifflin-St Jeor (más precisa que Harris-Benedict para atletas)

BMR = 10 × peso_kg + 6.25 × altura_cm − 5 × edad + 5

Ejemplo con Alejandro (93.6 kg, 193 cm, 27 años):
BMR = 10×93.6 + 6.25×193 − 5×27 + 5 = 936 + 1206.25 − 135 + 5 = 2012 kcal
PASO 2 — TDEE (gasto total diario)
TDEE = BMR × Factor de actividad

Multiplicadores:
· Sedentario (0-1 días/semana): × 1.2
· Ligeramente activo (1-3 días): × 1.375
· Moderadamente activo (3-5 días): × 1.55
· Muy activo (6 días, hipertrofia + cardio): × 1.725 ← tu caso
· Atleta de élite (2x/día): × 1.9

Alejandro: TDEE = 2012 × 1.725 = ~3471 kcal/día de mantenimiento
PASO 3 — Objetivo calórico por fase
DEFINICIÓN: TDEE − 400 a − 600 kcal/día
→ Máximo déficit recomendado para preservar músculo: −500 kcal/día
→ Con Alejandro: 3471 − 500 = ~2971 kcal/día
→ Pérdida esperada: ~0.5 kg/semana = ~2 kg/mes (teórico)
→ En la práctica sale ~1.3 kg/mes (retención agua, creatina, variabilidad)

VOLUMEN: TDEE + 200 a + 400 kcal/día
→ Superávit limpio para minimizar grasa: +300 kcal/día
→ Con Alejandro: 3471 + 300 = ~3771 kcal/día
→ Ganancia esperada: ~0.3 kg/semana = ~1.2 kg/mes

MANTENIMIENTO: TDEE exacto = ~3471 kcal/día
PASO 4 — Distribución de macros
Criterio: endo-mesomorfo + hipertrofia + 6 días/semana

PROTEÍNA (prioridad absoluta, antikatabolismo en déficit):
· Definición: 2.4 g × kg_peso_magro (~75% del peso total)
· Volumen: 2.0 g × kg_peso_magro
· Mantenimiento: 2.2 g × kg_peso_magro
→ Ej. def 93.6 kg → peso magro ~76 kg → 76 × 2.4 = ~182 g proteína/día

GRASAS (mínimo para función hormonal y salud):
· Todas las fases: 0.9–1.0 g × kg_peso
→ Ej. 93.6 × 0.9 = ~84 g grasa/día

HIDRATOS (variable de ajuste — lo que "queda" de calorías):
· HC = (Kcal_objetivo − Proteína×4 − Grasa×9) ÷ 4
→ Def: (2971 − 182×4 − 84×9) ÷ 4 = (2971 − 728 − 756) ÷ 4 = 1487÷4 = ~372 g HC/día
→ Vol: (3771 − 174×4 − 93×9) ÷ 4 = ~487 g HC/día
PASO 5 — Conversión macros → gramos de alimento
Cada alimento tiene un perfil nutricional conocido (base de datos abajo).
Se resuelve un sistema: dado que queremos X g de HC repartidos en comidas,
¿cuántos gramos de patata/arroz/pasta necesito?

Ejemplo comida (50% de HC diarios):
HC_comida = 372 × 0.5 = 186 g HC
Si elige patata (17g HC/100g crudo): 186 ÷ 0.17 = ~1094g patata → demasiado volumen
Con arroz integral (75g HC/100g crudo): 186 ÷ 0.75 = ~248g arroz en crudo
En la práctica se combina: ~75g arroz + ~150g patata = ~56+26 = 82g HC (una comida razonable)

El ajuste fino histórico (lo que hace tu nutricionista) está dentro de estos rangos
y es la combinación óptima para tu adherencia y digestión.

Integración con GitHub
¿Para qué sirve? Cada vez que calcules un nuevo mes, los datos se guardan automáticamente como un archivo JSON en tu repositorio de GitHub. El historial de commits se convierte en el historial de tu plan nutricional. Puedes acceder a cualquier mes anterior, comparar, y la herramienta siempre carga los datos más recientes.
Requisito: Un Personal Access Token de GitHub con permiso repo (o contents:write en fine-grained tokens). Nunca compartas este token. Si el repo es público, el JSON de tus dietas será visible — usa un repo privado.

🔧 Qué hace el código exactamente
Cuando pulsas "Guardar en GitHub" (tras añadir un mes), la herramienta llama a la API de GitHub:
PUT https://api.github.com/repos/{user}/{repo}/contents/dietas/{mes}.json

Headers:
Authorization: Bearer {token}
Content-Type: application/json

Body:
{
"message": "Añadir dieta {mes}",
"content": btoa(JSON.stringify(dietaData)) // base64
}
Al cargar la página, también hace un GET al repo para recuperar todos los JSON de /dietas/ y reconstruye la tabla. Así la herramienta funciona aunque borres el historial local.
